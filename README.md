## Designed for Human and GAI Use

These specifications are written in [**Markdown format**](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax) and made freely available on Github for these reasons:

- **Accessibility**: Markdown format is easily "understood" by generative AI tools, as the hierarchy and emphasis of content is clearly communicated through symbols. Content written in Markdown can also be displayed as formatted text on the web. Therefore, Markdown simultaneously optimizes content for people and LLMs.
- **Transparency & Efficiency**: When hosted on git-based systems, contributors, change logs, and version control happens automatically.

As Andrej Karpathy noted in his 2025 [Y Combinator talk](https://youtu.be/LCEmiRjPEtQ?si=mF66ukvERR6BPUc7), writing instructional content in Markdown is becoming a best practice.
